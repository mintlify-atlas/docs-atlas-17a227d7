Any authenticated user can file a report against a listing they believe is inappropriate, misleading, or a duplicate. Admins review reports and update their status as they work through the moderation queue.

<Note>
  Any authenticated user with the `buyer`, `seller`, or `admin` role can **create** a report. Only users with the `admin` role can **list** reports or **update** their status.
</Note>

**Report status workflow:**

```
open → reviewing → resolved
                 → dismissed
```

| Status | Meaning |
|---|---|
| `open` | Newly submitted, not yet reviewed |
| `reviewing` | An admin has started investigating |
| `resolved` | Admin took action (e.g., removed the listing) |
| `dismissed` | Admin reviewed and found no violation |

---

## Create a report

Submit a report against a listing. Requires authentication.

```
POST /v1/reports
```

**Required roles:** `buyer`, `seller`, `admin`

### Request body

<ParamField body="listingId" type="string" required>
  UUID of the listing being reported.
</ParamField>

<ParamField body="reporterId" type="string" required>
  UUID of the user filing the report. Should match the authenticated user's ID.
</ParamField>

<ParamField body="reason" type="string" required>
  Description of the issue. Between 3 and 500 characters. Be specific to help admins make a decision, e.g. `"Listing is a duplicate of listing ID abc123"` or `"Price is misleading — item is damaged"`.
</ParamField>

### Response

Returns `201 Created` with the new report object.

<ResponseField name="data" type="object">
  <Expandable title="properties">
    <ResponseField name="id" type="string">UUID of the created report.</ResponseField>
    <ResponseField name="listingId" type="string">UUID of the reported listing.</ResponseField>
    <ResponseField name="reporterId" type="string">UUID of the user who filed the report.</ResponseField>
    <ResponseField name="reason" type="string">The submitted reason text.</ResponseField>
    <ResponseField name="status" type="string">Initial status: `open`.</ResponseField>
    <ResponseField name="createdAt" type="string">ISO 8601 timestamp when the report was filed.</ResponseField>
    <ResponseField name="updatedAt" type="string">ISO 8601 timestamp of the last status change.</ResponseField>
  </Expandable>
</ResponseField>

### Example

```bash
curl -X POST http://localhost:8082/v1/reports \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "listingId": "11111111-1111-1111-1111-111111111111",
    "reporterId": "aaaaaaaa-aaaa-aaaa-aaaa-aaaaaaaaaaaa",
    "reason": "Listing is duplicate/incomplete"
  }'
```

```json
{
  "data": {
    "id": "rrrrrrrr-rrrr-rrrr-rrrr-rrrrrrrrrrrr",
    "listingId": "11111111-1111-1111-1111-111111111111",
    "reporterId": "aaaaaaaa-aaaa-aaaa-aaaa-aaaaaaaaaaaa",
    "reason": "Listing is duplicate/incomplete",
    "status": "open",
    "createdAt": "2025-10-29T22:41:17Z",
    "updatedAt": "2025-10-29T22:41:17Z"
  }
}
```

## List reports

Return a paginated list of all reports. Admin only.

```
GET /v1/reports
```

**Required roles:** `admin`

### Query parameters

<ParamField query="status" type="string">
  Filter by report status. One of: `open`, `reviewing`, `resolved`, `dismissed`. Omit to return reports in all statuses.
</ParamField>

<ParamField query="limit" type="number" default="20">
  Maximum number of results to return.
</ParamField>

<ParamField query="offset" type="number" default="0">
  Number of results to skip for pagination.
</ParamField>

### Response

<ResponseField name="data" type="object">
  <Expandable title="properties">
    <ResponseField name="items" type="object[]">
      Array of report objects.
    </ResponseField>
    <ResponseField name="total" type="number">
      Total number of reports matching the query.
    </ResponseField>
    <ResponseField name="limit" type="number">
      The limit value used for this response.
    </ResponseField>
    <ResponseField name="offset" type="number">
      The offset value used for this response.
    </ResponseField>
  </Expandable>
</ResponseField>

### Example

```bash
curl "http://localhost:8082/v1/reports?status=open" \
  -H "Authorization: Bearer $ADMIN_JWT_TOKEN"
```

```json
{
  "data": {
    "items": [
      {
        "id": "rrrrrrrr-rrrr-rrrr-rrrr-rrrrrrrrrrrr",
        "listingId": "11111111-1111-1111-1111-111111111111",
        "reporterId": "aaaaaaaa-aaaa-aaaa-aaaa-aaaaaaaaaaaa",
        "reason": "Listing is duplicate/incomplete",
        "status": "open",
        "createdAt": "2025-10-29T22:41:17Z",
        "updatedAt": "2025-10-29T22:41:17Z"
      }
    ],
    "total": 1,
    "limit": 20,
    "offset": 0
  }
}
```

## Update report status

Move a report through the moderation workflow. Admin only.

```
PATCH /v1/reports/:id/status
```

**Required roles:** `admin`

### Path parameters

<ParamField path="id" type="string" required>
  UUID of the report to update.
</ParamField>

### Request body

<ParamField body="status" type="string" required>
  New status for the report. One of: `open`, `reviewing`, `resolved`, `dismissed`.
</ParamField>

### Response

Returns `200 OK` with the updated report object.

### Example

```bash
curl -X PATCH http://localhost:8082/v1/reports/rrrrrrrr-rrrr-rrrr-rrrr-rrrrrrrrrrrr/status \
  -H "Authorization: Bearer $ADMIN_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"status": "resolved"}'
```

```json
{
  "data": {
    "id": "rrrrrrrr-rrrr-rrrr-rrrr-rrrrrrrrrrrr",
    "listingId": "11111111-1111-1111-1111-111111111111",
    "reporterId": "aaaaaaaa-aaaa-aaaa-aaaa-aaaaaaaaaaaa",
    "reason": "Listing is duplicate/incomplete",
    "status": "resolved",
    "createdAt": "2025-10-29T22:41:17Z",
    "updatedAt": "2025-10-30T10:15:00Z"
  }
}
```
