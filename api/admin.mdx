<Warning>
  All endpoints on this page require a JWT token from an account with the `admin` role. Requests from non-admin accounts will receive a `403 Forbidden` response.
</Warning>

## Creating admin accounts

Admin accounts cannot be self-registered through the public sign-up flow in a production environment. To create an admin user, insert the record directly into the database using a script such as `add_admin_user.sql`, or update an existing user's `role` column:

```sql
UPDATE users SET role = 'admin' WHERE email = 'admin@sjsu.edu';
```

## Get platform metrics

Return aggregate statistics about the platform. Useful for monitoring overall health and activity.

```
GET /v1/admin/metrics
```

**Required roles:** `admin`

### Response

<ResponseField name="data" type="object">
  <Expandable title="properties">
    <ResponseField name="listings" type="number">
      Total number of listings across all statuses.
    </ResponseField>
    <ResponseField name="users" type="number">
      Total number of registered user accounts.
    </ResponseField>
    <ResponseField name="reports" type="object">
      Report counts broken down by status.
      <Expandable title="properties">
        <ResponseField name="open" type="number">
          Number of reports with status `open`.
        </ResponseField>
        <ResponseField name="reviewing" type="number">
          Number of reports with status `reviewing`.
        </ResponseField>
        <ResponseField name="resolved" type="number">
          Number of reports with status `resolved`.
        </ResponseField>
        <ResponseField name="dismissed" type="number">
          Number of reports with status `dismissed`.
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

### Example

```bash
curl http://localhost:8082/v1/admin/metrics \
  -H "Authorization: Bearer $ADMIN_JWT_TOKEN"
```

```json
{
  "data": {
    "listings": 126,
    "users": 142,
    "reports": {
      "open": 3,
      "reviewing": 1,
      "resolved": 12,
      "dismissed": 4
    }
  }
}
```

## List all users

Return a paginated list of all registered user accounts.

```
GET /v1/admin/users
```

**Required roles:** `admin`

### Query parameters

<ParamField query="limit" type="number" default="20">
  Maximum number of users to return.
</ParamField>

<ParamField query="offset" type="number" default="0">
  Number of records to skip for pagination.
</ParamField>

### Response

<ResponseField name="data" type="object">
  <Expandable title="properties">
    <ResponseField name="items" type="object[]">
      Array of user objects.
      <Expandable title="user properties">
        <ResponseField name="id" type="string">UUID of the user.</ResponseField>
        <ResponseField name="name" type="string">Display name.</ResponseField>
        <ResponseField name="email" type="string">Email address.</ResponseField>
        <ResponseField name="role" type="string">Account role: `buyer`, `seller`, or `admin`.</ResponseField>
        <ResponseField name="createdAt" type="string">ISO 8601 registration timestamp.</ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name="total" type="number">
      Total number of users (before pagination).
    </ResponseField>
    <ResponseField name="limit" type="number">
      The limit value used for this response.
    </ResponseField>
    <ResponseField name="offset" type="number">
      The offset value used for this response.
    </ResponseField>
  </Expandable>
</ResponseField>

### Example

```bash
curl "http://localhost:8082/v1/admin/users?limit=20&offset=0" \
  -H "Authorization: Bearer $ADMIN_JWT_TOKEN"
```

```json
{
  "data": {
    "items": [
      {
        "id": "aaaaaaaa-aaaa-aaaa-aaaa-aaaaaaaaaaaa",
        "name": "Alice",
        "email": "alice@sjsu.edu",
        "role": "seller",
        "createdAt": "2025-10-29T22:41:17Z"
      },
      {
        "id": "bbbbbbbb-bbbb-bbbb-bbbb-bbbbbbbbbbbb",
        "name": "Bob",
        "email": "bob@sjsu.edu",
        "role": "buyer",
        "createdAt": "2025-10-30T08:00:00Z"
      }
    ],
    "total": 142,
    "limit": 20,
    "offset": 0
  }
}
```

## Force remove a listing

Immediately set a listing's status to `removed` regardless of its current state or owner. Use this to take down listings that violate platform policies.

```
POST /v1/admin/listings/:id/remove
```

**Required roles:** `admin`

<Warning>
  This action cannot be undone through the API. The listing record is soft-deleted (status set to `removed`) and will no longer appear in public browsing results.
</Warning>

### Path parameters

<ParamField path="id" type="string" required>
  UUID of the listing to remove.
</ParamField>

### Response

Returns `200 OK` with a confirmation object.

<ResponseField name="data" type="object">
  <Expandable title="properties">
    <ResponseField name="ok" type="boolean">
      `true` when the listing was successfully removed.
    </ResponseField>
  </Expandable>
</ResponseField>

### Example

```bash
curl -X POST http://localhost:8082/v1/admin/listings/11111111-1111-1111-1111-111111111111/remove \
  -H "Authorization: Bearer $ADMIN_JWT_TOKEN"
```

```json
{
  "data": {
    "ok": true
  }
}
```
